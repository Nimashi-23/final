Team

Technical Description

  • Site Structure: The layout utilizes \div\ tags, team photos are placed with \img\ tags, the names and Ids of the members are in \p\ tags, and the main and sub headings are assigned as \h1\ and \h2\ respectively.
  • Site Style: Team card \div\ elements are arranged in rows with the use of flexbox; the background image is set to have an ocean theme. Styling elements include modern \box-shadow\ and \border-radius\.
  • Additional Style: The text color and font weight of team members increase the readability while hover : active state allows for basic interactivity.
  • Responsive Design: Use of @media queries gives flexibility and ensures proper functionality and layout from lower to higher resolutions.

Accessibility

  • Semantic HTML: Assigns appropriate tags such as headings and paragraphs to maximize screen reader use.
  • Image Alt Text: Allows users with visual disabilities to comprehend the photos of team members.
  • Color Contrast: Using white letters against a dark background increases legibility.
  • Keyboard Navigation: Interactive buttons and objects are focusable enabling users working with keyboards to navigate easily.
  • Responsive Design: Addressing the needs of various devices and screen sizes.

Content Page

Technical Description

  1. Key HTML Elements:
    • (header): Displays the main title.
    • (nav): Fixed navigation bar for easy access.
    • (section) & (div): Organizes content into structured blocks.
    • (img): Background and content ima(/li)ges.
    • (a): Hyperlinks for navigation.
  2. Key CSS Techniques:
    • Background: background-image: cover; for a full-screen ocean theme.
    • Text Styling: color: white; text-align: center; for readability.
    • Navigation Bar: position: fixed; background-color: rgba(0, 0, 139, 0.8); for visibility.
    • Content Box: background-color: rgba(0, 0, 0, 0.7); to improve contrast.
    • Hover Effects: transition: 0.3s ease-in-out; for interactivity.

Accessibility

  • Semantic HTML – Used (header), (nav), (main), (section), and (footer) for better screen reader navigation.
  • Keyboard Navigation – Ensured logical tab order and added :focus styles for visibility.
  • High Contrast & Readability – Used accessible color contrast, readable fonts, and proper spacing.
  • Alt Text for Images – Ensured meaningful alt attributes (if images were included).
  • ARIA Attributes – Added aria-label for better navigation and assistive tech support.

Challenges and Lessons Learned

    Common Web Development Challenges:
  • Responsiveness: Making the page appear well on all devices usually involves techniques such as media queries in CSS.
  • Browser Compatibility: Browsers read code differently, hence the need to test across browsers.
  • Performance: Heavy images or heavy scripts can make the page slow, hence the need to optimize files and minimize server requests.
  • Security: Protection against threats such as XSS needs secure coding practices, e.g., input sanitization.
  • SEO: To ensure that a page ranks well, proper meta tags, keywords, and content optimization are to be used.
    • Key Lessons Learned:
  • Responsive Design: Flexbox, CSS Grid, and relative units enable a page to be responsive.
  • Cross-Browser Testing: BrowserStack sorts out compatibility issues.
  • Performance Optimization: Image compression, file minification, and caching enable a page to load faster.
  • Security Best Practices: Making websites secure involves input validation, HTTPS, and access control.
  • SEO Fundamentals: Correct content organization, mobile friendliness, and SEO tool utilization increase rankings.

Compliance

  • Data Protection & Privacy: Ensure GDPR compliance by informing users about data collection and providing a privacy policy.
  • Accessibility: Follow WCAG guidelines by using semantic HTML, alt text, keyboard navigation, and proper color contrast.
  • Security: Use HTTPS, strong passwords, and keep software updated to protect sensitive information.
  • Intellectual Property: Use licensed or free-to-use media and provide proper attribution.
  • Network Use: Ensure content doesn't violate Janet’s acceptable use policies, avoiding harmful activities like phishing.
