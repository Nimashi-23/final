HTML Structure

    1. display: flex: Used to align elements horizontally, especially for profile info and skills.
    2. justify-content: space-between: Helps distribute content evenly
    3. align-items: center: Ensures vertical alignment.
    Typography & Readability
    1. font-family: Likely a modern sans-serif for readability.
    2. color: White text on dark backgrounds for contrast.
    3. text-align: center: Used in profile sections for alignment.
    Skill Progress Bars
    1. width: X%: Dynamically adjusts the bar length.
    2. background-color: blue: Represents the progress visually.
    Form Styling
    1. input[type="text"], input[type="email"], textarea: Styled with padding and borders for better user experience.
    2. button:hover: Green button likely changes shade on hover.

    2. High Contrast & Readability: White text on black background for better visibility.

      Sitemap Page

      Technical Description

      HTML Structure

      1. div Containers: Used to structure different sections of the site map, such as the background, the site map nodes, and connecting lines.
      2. h1 Tag: Used for the title "Site Map," ensuring semantic meaning and proper heading hierarchy.
      3. VG or canvas: Used for drawing the connecting lines between nodes, ensuring smooth and scalable rendering.
      4. ul and li (Optional): If the site map is structured as a nested list, ul provides a semantic and hierarchical way to structure content.

        5. CSS Styling

          Background Styling
          1. background-image: url('underwater-image.jpg'):Sets the oceanic background with fish.
          2. background-size: cover:Ensures the background image covers the entire viewport.
          3. opacity: 0.8; (if used in a pseudo-element):Softens the background without affecting the readability of text.
          Text Styling
          1. font-size: 2em; font-weight: bold; color: white; text-align: center :Makes the site map title stand out.
          2. font-family: sans-serif:Ensures readability.
          Node Styling
          1. display: flex; justify-content: center; align-items: center :Centers the nodes within their respective divs.
          2. border-radius: 50%; background-color: green :Creates circular nodes.
          3. width: 80px; height: 80px :Maintains uniform sizing for each node.
          Flexbox or Grid for Layout
          1. display: flex; flex-direction: column; align-items: center :Aligns the nodes in a vertical tree structure.
          2. display: grid; (optional alternative):Used for a structured placement of nodes.

          Accessibility

          1. Semantic HTML - Used h1, nav, and ul for better screen reader navigation.
          2. Color Contrast - Ensured high contrast between text and background for readability.
          3. Keyboard Navigation - Used tabindex="0" for focusable elements and outline styles for visibility.
          4. ARIA Attributes - Added aria-label, role="tree", and aria-expanded for assistive technologies.

                          Challenges and Lessons Learned

                          • Challenges Encountered:
                            1. Slow Loading Times: The webpage took too long to load, especially on mobile.
                              Resolution: Optimized images, minified CSS/JS, and used lazy loading.
                            2. Inconsistent UI Elements: Buttons and forms appeared differently across devices.
                              Resolution: Used a design system with standardized styles and components
                            3. Accessibility Issues: Screen readers had trouble interpreting the content.
                              Resolution: Added ARIA labels, improved color contrast, and ensured proper heading structure.
                            4. Form Validation Problems: Users could submit incomplete or incorrect forms.
                              Resolution: Implemented JavaScript validation and backend verification.
                            5. Dynamic Content Handling: Updating sections dynamically without reloading was tricky. Resolution: Used JavaScript (AJAX) to update content smoothly.
                          • Lessons Learned:
                            1. Performance Matters: Optimizing assets significantly improves user experience.
                            2. Consistency is Key: Using a design system ensures uniformity across all pages.
                            3. Prioritize Accessibility: A well-structured site benefits all users, including those with disabilities.
                            4. User Feedback is Valuable: Testing with real users helps catch usability issues.
                            5. Iterate and Improve: Continuous testing and refining lead to a polished final product.

                          Compliance

                          1. Accessibility:
                            Implemented WCAG standards by incorporating descriptive alt text for images, maintaining high color contrast for readability, and structuring the site with semantic HTML to support screen readers.
                          2. Data Protection & Privacy:
                            Currently, no personal data is collected. Any future data collection will require explicit user consent, with a transparent privacy policy explaining data usage and storage practices.
                          3. Security Measures:
                            Secured the website using HTTPS to encrypt data transmission. Followed best practices, such as input validation and security headers, to mitigate vulnerabilities.
                          4. Intellectual Property & Content Rights:
                            Ensured that all content was either original, properly licensed, or credited to the rightful owners, adhering to copyright laws and fair usage guidelines.
