Splash Page

Technical Description

HTML Structure

  1. Header (h1) - Used for the title "The Ocean's Pulse" to ensure prominence and clear hierarchy.
  2. Paragraph (p) - Used for the mission statement to describe the purpose of the project.
  3. Bold Text (strong or b) - Emphasizes the "Group Members" heading for clear readability.
  4. Hyperlinks (a) - Group members' names are likely wrapped in a tags, styled as links for potential navigation.
  5. Image/Background (img or CSS background-image) - A full-screen oceanic background enhances the thematic experience.
  6. Loading Indicator (div or span) - Used to display "Loading, please wait..." with an animated spinner.

CSS Styling

  1. Background Styling
    • background-image: url(...) - Applies an immersive oceanic background.
    • background-size: cover; - Ensures the background covers the entire viewport.
    • background-position: center; - Keeps the image properly aligned.
  2. Typography
    • color: white; - Ensures readability against the dark background.
    • text-align: center; - Centers text elements for a balanced design.
    • font-weight: bold; - Used for headings and important text.
  3. Hyperlink Styling
    • color: blue; - Default hyperlink color for easy identification.
    • text-decoration: none; - Likely used to remove underlines.
    • hover effect - Could be applied to change color when hovered.
  4. Loading Animation
    • CSS Animations (@keyframes) - May be used to create a spinning effect.
    • position: absolute; bottom: 10px; - Places the loading text at the bottom.

Accessibility

  • High Contrast: White text on a dark background ensures readability.
  • Semantic HTML: Proper headings (h1), paragraphs (p), and emphasis (strong) aid screen readers.
  • Accessible Links: Descriptive anchor text and keyboard-friendly navigation enhance usability.
  • Loading Indicator: aria-live="polite" and role="status" ensure assistive technologies announce loading states.
  • Responsive Design: Scalable fonts and viewport settings make content adaptable.
  • Keyboard Support: Interactive elements are navigable via Tab and Enter.
  • Image Accessibility: alt attributes describe images or mark them decorative if unnecessary.

Volunteer Page

Technical Description

  1. Semantic HTML
    • Used (nav), (section), (article), (button), and (label) for screen reader compatibility.
    • Proper (label) elements and aria-label attributes for form fields.
  2. Keyboard Navigation & Focus
    • Logical tab order ensures smooth keyboard navigation.
    • :focus styles highlight elements for keyboard users.
    • "Skip to Content" link for quick access.
  3. Color Contrast & Readability
    • Ensured 4.5:1 contrast ratio for text.
    • alt attributes for images/icons for screen readers.
    • Clear sans-serif fonts with proper spacing.
  4. Forms & Feedback Section
    • Labels and error messages with sufficient contrast.
    • Emoji ratings include aria-labels for screen readers.
  5. Responsive & Adaptive Design
    • meta viewport ensures proper mobile scaling.
    • Touch-friendly buttons (min. 44x44px size).
    • No reliance on hover effects for key interactions.

Accessibility

  • Semantic HTML: Used section, button, and label for screen readers.
  • Keyboard Navigation: Logical tab order, :focus styling, and a "Skip to Content" link.
  • Contrast & Readability: 4.5:1 text contrast, alt attributes for images, and clear fonts.
  • Forms & Feedback: Proper labels, accessible error messages, and aria-labels for emoji ratings.
  • Responsive Design: meta viewport, touch-friendly buttons (44x44px), no hover-dependent actions.
  • ARIA Enhancements: aria-hidden for icons, role="alert" for messages, aria-live="polite" for updates.
  • Testing: Screen reader (NVDA, VoiceOver), keyboard navigation, contrast checker, and mobile-friendly layout.

Challenges and Lessons Learned

  • Challenges Encountered:
    1. Responsive Design: Ensuring the site looked good on different devices was challenging.
      Resolution: Used media queries to adjust layout and sizing for various screen sizes.
    2. Cross-Browser Compatibility: The page didn't display correctly on all browsers.
      Resolution: Applied browser-specific prefixes and tested across multiple browsers.
    3. Alignment and Layout: Achieving consistent spacing and positioning was difficult.
      Resolution: Used Flexbox and Grid Layout for better control over elements.
    4. Navigation Bar Issues: The mobile navigation was overlapping or disappearing.
      Resolution: Implemented a mobile-friendly hamburger menu with JavaScript.
    5. Typography: Fonts didn't render well, and text was hard to read.
      Resolution: Chose web-safe fonts and used responsive units for better readability.
  • Lessons Learned:
    1. Testing: Ensuring the site works on different devices and browsers is crucial.
    2. Flexibility with Layouts: CSS Grid and Flexbox are key for responsive designs.
    3. Performance Optimization: Optimizing images improves page load speed.
    4. Mobile-First Design: Media queries ensure a better mobile experience.
    5. Patience: Small issues require attention to detail and patience to resolve.

Compliance

  1. Accessibility:
    Ensured the site follows WCAG guidelines, with alt text for images, good color contrast, and semantic HTML for screen readers.
  2. Data Protection and Privacy:
    No personal data is collected. For future data collection, explicit consent would be obtained, with a privacy policy outlining data use.
  3. Security:
    Used HTTPS for secure communication and followed basic security practices to prevent vulnerabilities.
  4. Content and Copyright Compliance:
    Ensured all content was original or properly licensed, with appropriate attribution for external resources.
